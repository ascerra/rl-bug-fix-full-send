The script constructs temporary paths for local OCI layouts and unpacked directories using image labels (`com.redhat.component`, `version`, `release`). If two distinct images within a build share identical labels, the parallel processing tasks attempt to write to the same filesystem path concurrently, leading to conflicts and unpredictable failures.

Additionally, the original script had two other weaknesses: it did not guarantee the cleanup of these temporary artifacts if a command failed mid-execution, potentially leading to resource leaks on the runner.
